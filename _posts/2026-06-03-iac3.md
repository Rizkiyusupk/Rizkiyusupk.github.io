Jika sudah masuk ke tahap selanjutnya yaitu menjalankan script,jika di projek sebelumnya menggunakan salah satu fitur virtualbox yaitu shared folder kali ini saya akan menggunakan github

membuat repository baru untuk scrip ini jika sudah buat file dan isi file dengan shell ini

```
#!/bin/bash

useradd -m -s /bin/bash vagrant
echo "vagrant ALL=(ALL) NOPASSWD:ALL" > /etc/sudoers.d/vagrant

mkdir -p /home/vagrant/.ssh
curl -o /home/vagrant/.ssh/authorized_keys https://raw.githubusercontent.com/hashicorp/vagrant/main/keys/vagrant.pub

chown -R vagrant:vagrant /home/vagrant/.ssh
chmod 700 /home/vagrant/.ssh
chmod 600 /home/vagrant/.ssh/authorized_keys
```

```
- name: update package

  tasks:
    - name: update package

    - name: install openjdk
```


masuk ke playbook selanjutnya

```
vim playbook-install-jenkins-baremetal.yaml
|
- name: install jenkins
  hosts: Jenkins
  become: yes

  tasks:
    - name: adding key
      ansible.builtin.shell: curl -fsSL https://pkg.jenkins.io/debian-stable/jenkins.io-2026.key | tee /etc/apt/keyrings/jenkins-keyring.asc  > /dev/null

    - name: adding repository
      ansible.builtin.shell: echo "deb [signed-by=/etc/apt/keyrings/jenkins-keyring.asc]" https://pkg.jenkins.io/debian binary/ | tee /etc/apt/sources.list.d/jenkins.list > /dev/null

    - name: update package
      ansible.builtin.command: apt update
    - name: install jenkins
      ansible.builtin.shell: apt install jenkins -y
```

saya menggunakan jenkins versi terbaru,dan ada beberapa penyesuaian dalam playbook kali ini seperti memperbaiki command yang di gunakan, jika sebelum-sebelumnya saya menambahkan
key ke **/usr/share** tetapi apt mencari key di **/etc/apt** ini menimbulkan kekeliruan dan error yang fatal,ternyata saya tidak teliti dalam membuat script jadi saya memperbaiki dari
projek **buiild hybrid Ci/Cd**, jika sudah langsung gunakan command

```
    - name: make sure jenkins start
      ansible.builtin.systemd_service:
        state: started
```
