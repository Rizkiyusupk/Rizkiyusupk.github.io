### Overview 

Kali ini saya akan membahas projek lanjutan untuk series iac yang kali ini akan masuk ke tahap kedua dari projek iac ini, jika kemarin saya membahas tentang 
IaC: Building a Multi-Node Kubernetes Cluster from Scratch using Ansible, kali ini saya akan membahas fase kedua dari series iac ini, kemarin di projek iac yang pertama 
saya membuat cluster kubernetes dari awal full menggunakan ansible sebagai iac tools namun untuk bagian setup vm masih manual dan tidak otomatis,
dan kelanjutannya untuk projek kedua itu setup vm akan menjadi otomatis menggunakan tools vagrant sebagai provisioning, nantinya saya tidak perlu setup vm secara manual lagi
bisa saja saya menggunakan tools lain seperti terraform ataupun packer tapi ada kekurangan dalam infra saya, saya masih menggunakan local dan tidak menggunakan cloud ataupun 
proxmox, saya tidak menggunakan cloud karena tida mempunyai uang untuk membelina dan tidak menggunakan proxmox karena keterbatasan device jadi saya memutuskan untuk menggunakan vagrant 
sebagai provisioning, langsung saja masuk ke pembahasannya

### Setup Vagrant

Jika tahap pembuatan dari golden image sudah diselesaikan maka masuk ke tahap selanjutnya yaitu setup dan installasi vagrant yang akan di gunakan sebagai provisioning
untuk vm, pertama-tama matikan terlebih dahulu hyper-v nya  agar nantinya virtuabox tidak bentrok dengan vagrant dan membuat proses dari vagrant lambat dan tidak efisien
buka windows powoershell dan sebagai admin lalu ketik command ini

```
bcdedit /set hypervisorlaunchtype off
```

jika sudah tekan enter lalu tunggu

jika sudah pastikan outputnya seperti ini

```
PS C:\WINDOWS\system32>
>> bcdedit /set hypervisorlaunchtype off
The operation completed successfully.
```

nah untuk tahap selanjutnya yaitu restart untuk memboot ulang sistem,jika sudah di restart ketik command

```
systeminfo
```

pastikan tidak ada baris yang bertuliskan seperti inin

```
A hypervisor has been detected
```

jika sudah maka selamat untuk hyper-v telah di nonaktifkan jadinya vagrant bisa berjalan dengan lancar, untuk tahap selanjutnya yaitu installasi dari vagrant itu sendiri
masuk ke link [ini](/https://developer.hashicorp.com/vagrant/install) dan cari bagian untuk windows version lalu pilih yang amd64, tunggu sampai proses installasi selesai jika sudah 
klik hasil unduhan lalu install vagrant,jika sudah terinstall restert sekali lagi untuk memboot ulang sistem agar semua config terbaca, jika sudah masuk ke tahap berikutnya

### Provisioning Vm

Sampai juga di tahap provisioning vm yaitu tahap pembuatan vm yang tadinya di lakukan secara manual sekarang di lakukan secara otomatis menggunakan kode,di vagrant itu sendiri memiliki
file khusus untuk vagrant bisa membaca semua configurasi yang di tulis oleh user namanya vagrantfile,sebelum masuk ke vagrant file mari membuat folder dari projek disini saya membuat folder dengan nama learnt-vagrant buka vscode lalu open folder yang tadi baru saja di buat dengan nama learn-vagrant jika sudah buka terminal dari vscode dan ketik command

```
vagrant package --base nama-vm-yang-sudah-dibuat --output golden.box
|
vagrant package --base node-1 --output golden.box
```

tunggu sampai proses selesai lalu jika sudah buat file dengan nama vagrantfile lalu isi file dengan configurasi sebagai berikut

```
Vagrant.configure("2") do |config|
  config.vm.box = "golden.box"

  config.vm.define  "node-01" do |node|
    node.vm.hostname = "node-01"
    node.vm.provision "shell", name: "install-dependencies" , path: "install-dependencies.sh" 
    node.vm.provider "virtualbox" do |vb|
      vb.cpus = 2
      vb.memory = 2048 
    end
    node.vm.network  "private_network" , ip: "192.168.2.35"
  end
  config.vm.define  "node-02" do |node|
    node.vm.hostname = "node-02"
    node.vm.provider "virtualbox" do |vb|
      vb.cpus = 2
      vb.memory = 2048 
    end
    node.vm.network  "private_network", ip: "192.168.2.36"
  end
  config.vm.define  "node-03" do |node|
    node.vm.hostname = "node-03"
    node.vm.provider "virtualbox" do |vb|
      vb.cpus = 2
      vb.memory = 2048 
    end
    node.vm.network  "private_network" , ip: "192.168.2.37"
  end
    
end

```
dan siapkan dependencies yang berisi baris perintah untuk melakukan installasi package tertentu 

```
#!/bin/bash
apt-get update
apt-get install ansible -y
```

isi dari vagrantfile tersebut untuk bagian pertama vagrant.configure itu berarti versi dari vagrant ini merupakan 1.1+ jika saya menggunakan vagrant.configure 1 maka versi vagrant
yang saya gunakan itu 1 kebawah,karena saya menginstall versi terbaru saya menggunakan vagrant configure 2, untuk confg.vm menggunaka golden box yang baru saja dibuat sesuaikan dengan
nama golden box yang tadi dibuat,lalu masuk ke multi node configuration,di bagian pertama node-01 saya menamai vm dengan nama node-01 lalu saya menggunakan provision dan men set nya 
sebagai private provision karena jika saya menaruh baris provision di bagian luar multi node configuration, semua node akan terinstall file dependesi yang berisi installasi ansible,
karena ansible cuman butuh satu node controller dan tidak butuh agent atau agentless maka dari itu baris provision saya taruh hanya di baris config node-01,lalu saya menggunakan 
ip static untuk men set ip agar nantinya mudah untuk di debug, untuk selanjutnya saya menggunakan provider virtualbox dengan isi cpu 2 dan memory 2048, walaupun di golden image sudah di
tentukan tapi tetap saja vagrant tidak mengetahui jika vm sudah memiliki cpu dan memory,dan untuk sisanya sama saja,masuk ke dependensi untuk installasi ansible,lalu jika sudah 
ketik command

```
vagrant up
```

tunggu hingga vagrant selesai melakukan prosesnya
