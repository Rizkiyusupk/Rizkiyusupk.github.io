### Provisioning Vm

Sampai juga di tahap provisioning vm yaitu tahap pembuatan vm yang tadinya di lakukan secara manual sekarang di lakukan secara otomatis menggunakan kode,di vagrant itu sendiri memiliki
file khusus untuk vagrant bisa membaca semua configurasi yang di tulis oleh user namanya vagrantfile,sebelum masuk ke vagrant file mari membuat folder dari projek disini saya membuat folder dengan nama learnt-vagrant buka vscode lalu open folder yang tadi baru saja di buat dengan nama learn-vagrant jika sudah buka terminal dari vscode dan ketik command

```
vagrant package --base nama-vm-yang-sudah-dibuat --output golden.box
|
vagrant package --base node-1 --output golden.box
```

tunggu sampai proses selesai lalu jika sudah buat file dengan nama vagrantfile lalu isi file dengan configurasi sebagai berikut

```
Vagrant.configure("2") do |config|
  config.vm.box = "golden.box"

  config.vm.define  "node-01" do |node|
    node.vm.hostname = "node-01"
    node.vm.provision "shell", name: "install-dependencies" , path: "install-dependencies.sh" 
    node.vm.provider "virtualbox" do |vb|
      vb.cpus = 2
      vb.memory = 2048 
    end
    node.vm.network  "private_network" , ip: "192.168.2.35"
  end
  config.vm.define  "node-02" do |node|
    node.vm.hostname = "node-02"
    node.vm.provider "virtualbox" do |vb|
      vb.cpus = 2
      vb.memory = 2048 
    end
    node.vm.network  "private_network", ip: "192.168.2.36"
  end
  config.vm.define  "node-03" do |node|
    node.vm.hostname = "node-03"
    node.vm.provider "virtualbox" do |vb|
      vb.cpus = 2
      vb.memory = 2048 
    end
    node.vm.network  "private_network" , ip: "192.168.2.37"
  end
    
end

```
dan siapkan dependencies yang berisi baris perintah untuk melakukan installasi package tertentu 

```

apt-get update
apt-get install ansible -y
```

### Setup Ansible

Masuk ke tahap inti dari installasi kubernetes pertama-tama mari membuat sebuah ansible config

```
vim ansible.cfg
|
vagrant@node-01:~$ cat ansible.cfg 
[defaults]
inventory = ./inventory
remote_user = vagrant
host_key_checking = False
become = yes
become_method = sudo
become_ask_pass = False
```

ansible config berfungsi untuk memberitahu ansible lokasi dari inventory tapi bukan hanya itu saja ada beberapa fungsi lain dari ansible config ini seperti remote user 
privillage esallation , misalnya untuk remote user ketika akan menggunakan ansible cara kerja ansible itu menggunakan ssh sebagai cara kerja dari si ansible,
nah untuk remote user sendiri itu bisa di tentukan lewat ansible config dengan remote user,selain dari itu privillage escallation penting
untuk mengeksekusi task dan modul yang ada karena ada banyak task yang membutuhkan root permission jika saja tidak ada privillage escalltion 
tidak mungkin task yang memerlukan root permission bisa di eksekusi, **NOTE JIKA MENGGUNAKAN PRIVILLAGE ESCALATION PASTIKAN REMOTE USER YANG ADA DI TARGER SERVER SUDAH MASUK KE FILE SUDOERS!! GUNAKAN PERINTAH SUDO VISUDO AGAR BISA MENGEDIT SUDOERS**,
masuk ke tahap selanjutnya yaitu pembuatan inventory

```
vim inventory
|
vim inventory
|
[master]
node1 ansible_host=192.168.2.35

[slave1]
node2 ansible_host=192.168.2.36

[slave2]
node3 ansible_host=192.168.2.37



[all:vars]
[all:vars]
ansible_user=vagrant
ansible_ssh_private_key_file=~/.ssh/id_rsa

```

inventory berfungsi sebagai tempat penyimpanan informasi penting seperti grup dan hostname atau ip dari target server,
lihat contoh diatas ada baris dengan nama master , slave1,slave2 nah itu semua adalah grup yang bisa berisi ip dari target ataupun hostname dari target,
dan di inventory juga bisa menyimpan variable yang bisa di pakai kapan saja misal dalam eksekusi playbook,ingat dengan ansible config diatas 
ada baris dengan value seperti ini

```
./inventory
```

masuk ke playbook pertama,playbook kali ini akan menonaktifkan swap secara permanent,swap mati adalah salah satu syarat utama buat kubelet bisa berjalan buat file dengan nama

```
vim playbook-swap.yaml
|
- name: disable swap permanent
  hosts: slave1,slave2,master
  become: yes
  tasks:
    - name: comment out swap di fstab
      ansible.builtin.lineinfile:
        path: /etc/fstab
        regexp: '^/swap'
        line: '#/swap.img       none    swap    sw      0       0'
        state: present

    - name: disable swap sekarang
      ansible.builtin.command: swapoff -a
```

playbook diatas merupakan playbook untuk membuat swap mati secara permanent,dengan mencari regex dari path file tertentu misal dengan
menggunakan regexp kita bisa mencari pola baris tertentu dengan membuat baris 

```
regexp: '^/swap'
```

nah dalam baris file diatas itu ada tanda '^' nah tanda itu berarti cari baris dengan awalan seperti diatas dalam path yang sudah di tulis diatas
lalu ada baris

```
line: '#/swap.img       none    swap    sw      0       0'
```

baris diatas menandakan bahwa masukan sebelum pattern line ,lalu baris line itu maksudnya baris yang akan di berikan tanda '#' jadinya jika line pattern
yang ingin di masukan itu berupa baris swap maka baris swap itu yang akan diberikan tanda '#' 
trus untuk task selanjutnya itu gunakan swapoff -a sebagai pelengkap agar swap benar benar mati gunakan modul ansible.builtin.command agar bisa 
menggunakan perintah swapoff -a
jika sudah maka gunakan command

- name: do a configuration
  hosts: slave1,slave2,master

    - name: set module
      ansible.builtin.blockinfile:
        path: /etc/modules-load.d/k8s.conf

          overlay
          br_netfilter
        create: yes
    - name: load overlay
      ansible.builtin.command: modprobe overlay
    - name: load br_netfilter
      ansible.builtin.command: modprobe br_netfilter
    - name: set sysctl
      ansible.builtin.blockinfile:
        path: /etc/sysctl.d/k8s.conf
        block: |
          net.bridge.bridge-nf-call-iptables  = 1
          net.ipv4.ip_forward                 = 1
          net.bridge.bridge-nf-call-ip6tables = 1
        create: yes
    - name: load systctl
      ansible.builtin.command: sysctl --system

vim playbook-pkg.yaml
|
- name: install packages
  hosts: slave1,slave2,master
  become: yes
  

  tasks:
    - name: install package for containerd
      ansible.builtin.command: sudo apt install -y apt-transport-https ca-certificates curl gnupg lsb-release
    - name: add gpg key
      ansible.builtin.shell:     curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor --batch --yes -o /usr/share/keyrings/docker-archive-keyring.gpg

    - name: add gpg key1
      shell:  echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
    
    - name: update package
      ansible.builtin.shell: apt update 
    - name: install cotainerd
      ansible.builtin.shell: apt install containerd -y
    - name: create directory
      ansible.builtin.shell:  mkdir -p /etc/containerd
    - name: add config containerd
      ansible.builtin.shell: containerd config default | sudo tee /etc/containerd/config.toml

    - name: set cgroup
      ansible.builtin.shell: sudo sed -i 's/SystemdCgroup \= false/SystemdCgroup \= true/g' /etc/containerd/config.toml
    - name: Restart service cron on centos, in all cases, also issue daemon-reload to pick up config changes
      ansible.builtin.systemd_service:
      state: restarted 
        daemon_reload: true
        name: containerd
    - name: Enable service httpd and ensure it is not masked
      ansible.builtin.systemd_service:
        name: containerd
        enabled: true
        masked: no

```

vim playbook-join.yaml
|
- name: init kubernetes
  hosts: master
  become: yes
  tasks:
    - name: pre-pull kubernetes images
      ansible.builtin.command: kubeadm config images pull
      register: pull_output
    - name: init
      ansible.builtin.command: kubeadm init --apiserver-advertise-address=192.168.2.35 --pod-network-cidr=10.244.0.0/16 --node-name=node1
      register: kube_output
      async: 600
      poll: 30
      ignore_errors: yes

    - name: ambil join command
      ansible.builtin.shell: kubeadm token create --print-join-command
      register: join_cmd_output

    - name: set fact join command
      set_fact:
        join_command: "{{ join_cmd_output.stdout }}"

- name: join worker ke master
  hosts: slave1,slave2
  become: yes
  tasks:
    - name: join cluster
      ansible.builtin.shell: "{{ hostvars['node1']['join_command'] }}"
```

task pertama yaitu untuk menarik config image agar proses init tidak terlalu lama atau memakan banyak sekali waktu hingga bisa  time out dan proses init bisa eror,selanjutnya 
saya mengunakan builtin command untuk melakukan init di kubernetes karena saya sebelumnya sudah melakukan penarikan image agar proses init tidak memakan waktu terlalu lama,jadinya untuk
