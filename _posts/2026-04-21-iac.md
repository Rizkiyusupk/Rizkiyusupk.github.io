### Overview
Pada postingan kali ini saya akan membahas bagaimana penerapan infrastructure as code bekerja,
infrastructure as code atau disingkat sebagai iac adalah salah satu bentuk metodologi dalam dunia automation 
sebagai salah satu contoh kamu bisa aja mengotomatisasi semua proses konfigurasi yang tadinya manual jadi otomatis tanpa manual 
handling lagi, use casenya misalkan kamu punya lebih dari 100 node server yang harus di konfigurasikan akan memakan waktu yang sangat
lama untuk melakukan konfigurasi satu persatu terhadap semua node yang ada,maka dari itu lahirlah solusi yaitu automation iac,
dengan iac kamu bisa melakukan konfigurasi ke seluruh node yang ada tanpa harus melakukan manual handling satu persatu.Jadi iac lahir 
sebagai solusi dan iac juga bisa di terapkan ke semua model komputasi baik on-premise , cloud , ataupun container, langsung saja
masuk ke pembahasanya

masuk ke modul selanjutnya,modul kali ini akan menonaktifkan swap secara permanent,swap mati adalah salah satu syarat utama buat kubelet bisa berjalan buat file dengan nama

```
vim playbook-swap.yaml
|
- name: disable swap permanent
  hosts: slave1,slave2,master
  become: yes

  tasks:
    - name: disable swap
      ansible.builtin.lineinfile:
        regexp: '^/dev/mapper/rhel-swap'
        insertbefore: '^/dev/mapper/rhel-swap'
        line: '#/dev/mapper/rhel-swap'
        path: /etc/fstab
    - name: swap disable
      ansible.builtin.command: swapoff -a
```

playbook diatas merupakan playbook untuk membuat swap mati secara permanent,dengan mencari regex dari path file tertentu misal dengan
menggunakan regexp kita bisa mencari pola baris tertentu dengan membuat baris 

```
regexp  : '^line/pattern/'
```

nah dalam baris file diatas itu ada tanda '^' nah tanda itu berarti cari baris dengan awalan seperti diatas dalam path yang sudah di tulis diatas
lalu ada baris

```
insertbefore: '^line/pattern/'
line: '#/line/pattern'
```

baris diatas menandakan bahwa masukan sebelum pattern line ,lalu baris line itu maksudnya baris yang akan di berikan tanda '#' jadinya jika line pattern
yang ingin di masukan itu berupa baris swap maka baris swap itu yang akan diberikan tanda '#' 
trus untuk task selanjutnya itu gunakan swapoff -a sebagai pelengkap agar swap benar benar mati gunakan modul ansible.builtin.command agar bisa 
menggunakan perintah swapoff -a
jika sudah maka gunakan command

```
ansible-playbook playbook.yaml
|
ansible-playbook playbook-swap.yaml
```

masuk ke playbook untuk menambahkan dns hosts name,nah untuk lebih mempermudah node agar saling mengenal satu sama lain saya akan membuat sebuah playbook yang berfungsi untuk menambahkan dns hosts langsung saja masuk ke playbooknya

```
vim playbook-ip.yaml
|
- name: adding diffrent ip for node
  hosts: slave1,slave2,master
  become: yes

  tasks:
    - name: add ip
      ansible.builtin.blockinfile:
        path: /etc/hosts
        block: |
          192.168.1.126 node1
          192.168.1.127 node2
          192.168.1.128 node3
        create: yes

```

oke playbook selanjutnya bagian config untuk selinux,dan beberapa modul tambahan seperti set sysctl dan 
modprobe,langsung saja masuk ke playbooknya

```
vim playbook-config.yaml
|
- name: do a configuration
  hosts: slave1,slave2,master
  become: yes

  tasks:
    - name: set SELINUX
      ansible.builtin.lineinfile:
        path: /etc/selinux/config
        regexp: '^SELINUX='
        line: SELINUX=permissive
    - name: set module
      ansible.builtin.blockinfile:
        path: /etc/modules-load.d/k8s.conf
        block: |
          overlay
          br_netfilter
        create: yes
    - name: load overlay
      ansible.builtin.command: modprobe overlay
    - name: load br_netfilter
      ansible.builtin.command: modprobe br_netfilter
    - name: set sysctl
      ansible.builtin.blockinfile:
        path: /etc/sysctl.d/k8s.conf
        block: |
          net.bridge.bridge-nf-call-iptables  = 1
          net.ipv4.ip_forward                 = 1
          net.bridge.bridge-nf-call-ip6tables = 1
        create: yes
    - name: load systctl
      ansible.builtin.command: sysctl --system

```

di playbook ini saya mengggunakan tiga modul dalam satu play modul pertama itu line in file yang pertama yaitu line in file , block in file, dan 
yang terakhir itu builtin command,yang task pertama itu untuk mencari pattern file tertentu di path yang sudah di tentukan,misal yang ada di contoh itu mencari baris 
SELINUX di path /etc/selinux/config dan ubah filenya dengan menambahkan baris 'permissive' , dan untuk file task yang kedua itu block file seperti yang sudah di jelaskan

jika sudah maka masuk ke playbook selanjutnya yaitu installasi container runtime atau cri, karena saya  menggunakan crio seperti yang terteri di daftar tools diatas
saya akan menginstall crio,langsung saja masuk ke playbooknya

```
vim playbook-pkg.yaml
|
- name: install packages
  hosts: slave1,slave2,master
  become: yes
  vars:
    crio_version: "v1.30"

  tasks:
    - name: add repo
      ansible.builtin.blockinfile:
        path: /etc/yum.repos.d/cri-o.repo
        block: |
          [cri-o]
          name=CRI-O
          baseurl=https://pkgs.k8s.io/addons:/cri-o:/stable:/{{ crio_version }}/rpm/
          enabled=1
          gpgcheck=1
          gpgkey=https://pkgs.k8s.io/addons:/cri-o:/stable:/{{ crio_version }}/rpm/repodata/repomd.xml.key
        create: yes
    - name: install crio
      ansible.builtin.dnf:
        name: cri-o
        state: present
    - name: enable crio
      ansible.builtin.systemd:
        name: crio
        enabled: yes
        state: started

```

playbook diatas menggunakan 3 modul yaitu block in file,dnf untuk package manager dari rhel, lalu ada systemd untuk reload, untuk task pertama itu menambahkan repo 
resmi dari crio ke dalam node jangan lupa jika ingin versi tertentu buat terlebih dahulu vars di bagian awal dari playbooknya lalu untuk task kedua itu installasi crio
jika tadi sudah berhasil menambahkan repo resmi dari crio ke dalam node sekarang bagian installasi dari crio itu sendiri,gunakan modul dnf sebagai default package manager
dari rhel jika ubuntu berarti menggunakan apt,tergantung os yang di gunakan,lalu untuk task ketiga untuk menjalankan si crio agar bisa langsung berjalan, jalankan playbook
gunakan command

```
ansible-playbook playbook-pkg.yaml

- name: install kubernetes
  hosts: slave1,slave2,master
  become: yes
  vars:
    KUBERNETES_VERSION: "v1.30"

  tasks:
    - name: add repo
      ansible.builtin.blockinfile:
        path: /etc/yum.repos.d/kubernetes.repo
        block: |
          [kubernetes]
          name=Kubernetes
          baseurl=https://pkgs.k8s.io/core:/stable:/{{ KUBERNETES_VERSION }}/rpm/
          enabled=1
          gpgcheck=1
          gpgkey=https://pkgs.k8s.io/core:/stable:/{{ KUBERNETES_VERSION }}/rpm/repodata/repomd.xml.key
        create: yes
    - name: install kubernetes
      ansible.builtin.dnf:
        name: kubelet
        state: present
    - name: install kubernetes2
      ansible.builtin.dnf:
        name: kubectl
        state: present
    - name: install kubernetes3
      ansible.builtin.dnf:
        name: kubeadm
        state: present
    - name: enable kubelet
      ansible.builtin.systemd:
        state: started
        enabled: yes
        name: kubelet

```

sampai diakhir juga playbook terakhir yaitu join node langsung saja ke playbooknya

```
vim playbook-join.yaml
|
- name: init kubernetes
  hosts: master
  become: yes

  tasks:
    - name: init
      ansible.builtin.command: kubeadm init  --apiserver-advertise-address=192.168.1.xxx  --pod-network-cidr=10.244.0.0/16 --node-name=node1
      register: kube_output
      ignore_errors: yes
    - name: store the output
      ansible.builtin.copy:
        content: "{{ kube_output.stdout }}"
        dest: "/home/iki/token.txt"
    - name: Ambil join command secara bersih
      shell: echo "{{ kube_output.stdout }}" | grep -A 2 "kubeadm join" | tr -d '\\\n'
      register: clean_join
    - name: join command
      set_fact:
        join_command: "{{ clean_join.stdout }}"
- name: join
  hosts: slave1,slave2,
  become: yes

  tasks:
    - name: join command
      ansible.builtin.shell: "{{ hostvars['node1'].join_command }}"


```

nah untuk tahap ini ada beberapa penjelasan yang bisa saya sampaikan, pertama saya menggunakan module builtin command supaya saya bisa menrun command init 
dari master, lalu setelah itu output dari init itu di simpan kedalam sebuah file.txt ini sebenarnya opsional karena jaga jaga saja untuk menyimpan hasil dari init
yahh meskipun tinggal buat lagi tokennya hehehhe :v , lalu di task ketiga saya menggunakan module shell dan bukan command biasa karena modul shell mendukung adanya
pipe di dalamnya jika saja saya menggunakan modul command, nantinya ansible mengangap pipe sebagai teks biasa tapi di modul shell pipe akan di anggap sebagai wildcard
dan bisa gunakan seperti wildcard pada umumnya, nah untuk pipe nya sendiri itu mencari output dua yang memiliki kata kubeadm dan mencari dua baris kebawah mulai dari 
output 

```
kubeadm 
```

lalu hilangkan backslash nya '\\' karena nantinya python yang ada di ansible tidak bisa mengangap ini sebagai teks output dari init biasa melainkan suatu program aneh
dan akan memunculkan sebuah error, lalu setelah itu di task selanjutnya yaitu menaruh output tadi kedalam sebuah set_fact karena agar bisa di lihat oleh node lainnya
jadinya node lain bisa mencopy apa yang ada di dalam set_fact itu lalu di gunakan di play kedua , nah play kedua itu adalah join commandnya untuk di play ini khusus worker
saja, si worker akan bertanya ke master "apakah ada variable yang bernama join_command? jika ada aku mau menggunakannya!!" kurang lebih seperti itu dari task di play 2
