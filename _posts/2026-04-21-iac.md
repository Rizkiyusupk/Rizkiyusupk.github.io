### Overview
Pada postingan kali ini saya akan membahas bagaimana penerapan infrastructure as code bekerja,
infrastructure as code atau disingkat sebagai iac adalah salah satu bentuk metodologi dalam dunia automation 
sebagai salah satu contoh kamu bisa aja mengotomatisasi semua proses konfigurasi yang tadinya manual jadi otomatis tanpa manual 
handling lagi, use casenya misalkan kamu punya lebih dari 100 node server yang harus di konfigurasikan akan memakan waktu yang sangat
lama untuk melakukan konfigurasi satu persatu terhadap semua node yang ada,maka dari itu lahirlah solusi yaitu automation iac,
dengan iac kamu bisa melakukan konfigurasi ke seluruh node yang ada tanpa harus melakukan manual handling satu persatu.Jadi iac lahir 
sebagai solusi dan iac juga bisa di terapkan ke semua model komputasi baik on-premise , cloud , ataupun container, langsung saja
masuk ke pembahasanya

### Tools
- **Rhel (plow)** : 9.6
- **Kubernetes** : v1.30
- **Container Runtime Interface** : crio 1.30.10
- **CNI Plugin** : Flannel
- **Vmware Workstation** : 25h2
- **Ansible Core** : 2.14.18


| Alat | Kelebihan (+) | Kekurangan (-) |
| :--- | :--- | :--- |
| **Ansible** | Ringan, cepat, dan tidak butuh install software di target. | Bisa melambat jika jumlah server mencapai ribuan. |
| **Chef** | Sangat fleksibel untuk konfigurasi yang sangat kompleks. | Setup awal rumit dan butuh agent di setiap node. |
| **Puppet** | Sangat kuat dalam menjaga konsistensi kondisi server. | Kurang fleksibel untuk tugas yang sifatnya sekali jalan. |
