### **Jenkins tidak bisa di jangkau oleh webhook gitlab** 
Ketika jenkins node tidak bisa di akses oleh webhook gitlab maka keseluruhan workflow di projek ini akan berhenti total karena memang 
tujuan dari webhook di projek ini itu bertujuan agar nantinya triger pipeline lalu akan mendeploy secara otomatis ketika dev push dan webhook aktif yang akan menjalankan si jenkins file, nah ada momen dimana si webhook gitlab ini tidak bisa mengakses jenkins node dan akan keluar sebuah pesan eror seperti dibawah ini  

![logooo1](assets/images/errors/Screenshot 2026-03-03 214408.png)

nah eror diatas bisa terjadi karena si gitlab mencoba mengakses ke dalam jaringan node jenkins tetapi tidak bisa karena node belum memiliki public ip dan adanya firewall,untuk itu dibutuhkan ngrok agar nantinya bisa diakses oleh gitlab webhook.

### Solving Problem
Dikarenakan tadi sudah tau apa yang harus di lakukan maka sekarang tinggal eksekusi install ngrok,[install ngrok](https://ngrok.com/docs/guides/device-gateway/linux) 
nah jika sudah masuk ke terminal jenkins node lalu jalankan 

```
ngrok http port-jenkins
|
ngrok http 9091
```

kurang lebih seperti ini 

![logoo9](/assets/images/Screenshot 2025-12-10 170011.png)

nah jika sudah maka hasil webhook akan berhasil 

![logoo12](/assets/images/errors/Screenshot 2026-03-03 220030.png)

nah selesai sudah untuk problem dimana jenkins node tidak bisa diakses,next

### **Jenkins tidak bisa diakses melalui browser**
Untuk problem kedua ini jenkins tidak bisa diakses sama sekali lewat browser, jika kita ingin membuat sebuah
pipeline maka kita  perlu mengakses jenkins dashboard untuk bisa membuat sebuah pipeline maka dari itu wajib hukumnya
untuk bisa mengakses dashboard jenkins agar bisa membuat pipeline, seperti ini

![lgoo123](/assets/images/errors/jLFuJ.png)

gambar diatas hanyalah sebuah contoh yang di ambil dari internet,gambaran utamanya itu jika kamu akses jenkins melalui

```
ip-node:port-jenkins

```

maka seharusnya yang muncul itu ialah dashboard jenkins seperti ini

![lasdj](/assets/images/errors/Screenshot 2026-03-03 230657.png)

tapi kenapa ya jenkins dashboard tidak bisa diakses melalui browser,singkatnya ini karena peraturan ketat dari firewall yang akan mendump semua
proses 3 way handshake, yang dimana akan terjadinya pertukaran informasi 

```
syn

ack

```

![asb](/assets/images/errors/syn.png)

kurang lebih seperti itulah cara kerja dari pertukaran informasi di dalam komputer,nah dalam problem ini pertukaran informasi antara laptop pertama yang digunakan dan jenkins node atau laptop kedua yang di jadikan home server itu tidak berhasil secara sempurna ada kesalahan
dimana firewall tidak meloloskan syn-ack dari server ke client jadinya dashboard jenkins tidak bisa diakses,untuk masalah ini ada dua cara
yaitu dengan

### Solving problem
nah cara pertama itu dengan melakukan tunneling ke server di port dimana jenkins berjalan 

```
ssh -L 8888:localhost:9091 hostname@ip-node
|
ssh -L 8888:localhost:9091 rizky@192.168.100.7

```

dengan melakukan ini jenkins bisa langsung diakses dengan mudahnya dengan membuka browser lalu ketik

```
localhost:8888

```

ini beberapa contohnya 

![lasocj](/assets/images/errors/Screenshot 2026-03-02 200031.png)

dan ini dari browser


![asdjaisf](/assets/images/errors/Screenshot 2026-03-02 195808.png)

nah untuk selanjutnya yaitu cara kedua cara ini lebih permanen jadi anda tidak usah melakukan tunneling lagi 
jadi dengan menambah rule di port 9091 port jenkins berjalan

```
sudo firewall-cmd --zone=public --add-port=9091/tcp --permanent
sudo firewall-cmd --reload
```

copy lalu paste comamnd diatas untuk menambah rule agar port 9091 di pindahkan ke zona publlic atau jika ingin menambah suatu
interface juga bisa 

```
sudo firewall-cmd --zone=trusted --change-interface=wlo1 --permanent
sudo firewall-cmd --reload
```

### **Jenkins file tidak tereksekusi di pipeline**

nah untuk selanjutnya eror yang terjadi ini bisa terjadi ketika pipeline sudah berjalan namun jenkins file tidak tereksekusi jadinya 
deployment berhenti, contohnya seperti ini

![lasda](/assets/images/errors/Screenshot 2026-03-03 144328.png)

nah eror di atas bisa saja terjadi ketika user jenkins tidak memiliki permission untuk mengeksekusi jenkinsfile,maksudnya kube config yang di copy dari node master lalu di paste di user jenkins itu tidak memiliki permission yang sesuai dengan kebutuhan user jenkins,
atau bisa juga dengan tidak adanya kube config di user jenkins nantinya node jenkins kebingungan bagaimana caranya mengekeskusi jenkins file
tanpa adanya kube config
