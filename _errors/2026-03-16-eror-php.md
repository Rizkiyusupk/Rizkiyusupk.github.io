### Tidak bisa membuat image 

Dikarenakan di post deploy php dibutuhkan sebuah image untuk deploy dan imagenya itu di buat manual dan tidak menggunakan image dari registry
maka perlu untuk membuild semua dari awal nah proses build image ini sangat di perlukan karena nantinya kita akan membutuhkan imagenya untuk deploy,
dalam kasus ini prose ketika build itu tidak bisa di lakukan karena adanya eror yang terjadi ketika membuild image, saya akan contohkan seperti ini

```
docker build -t image .
|
docker build -t php-test2 .
```

### Mysql tidak terkoneksi dengan php
Untuk eror selanjutnya akan sedikit rumit karena ini berhubungan dengan network dan juga database karena eror kali ini adalah eror 
dimana mysql tidak terkoneksi dengan php dan malah akan muncul eror seperti ini

![asdihad](/assets/images/php1/Screenshot 2026-03-14 150626.png)

atau bahkan seperti ini

```
Warning: mysqli::__construct(): php_network_getaddresses: getaddrinfo for mysql failed: Temporary failure in name resolution in /var/www/html/index.php on line 7Fatal error: Uncaught mysqli_sql_exception: php_network_getaddresses: getaddrinfo for mysql failed: Temporary failure in name resolution in /var/www/html/index.php:7 Stack trace: #0 /var/www/html/index.php(7): mysqli->__construct('mysql', 'root', Object(SensitiveParameterValue), 'appdb') #1 {main} thrown in /var/www/html/index.php on line 7
```

output diatas merupakan contoh dari eror yang di sebabkan mysql yang tidak terkoneksi dengan php,itu terjadi karena flannel mencari lewat enp0s3 dan bukannya lewat enp0s8 yang merupakan ip jadinya jika flannel mencari di enp0s3 maka flannel cuman berputar putar karena bukan mencari ip 
melainkan malah berputar putar di nat , coba liat beberapa gambar ini 

![aisda](/assets/images/php1/Screenshot 2026-03-16 144931.png)

diatas dari node master

![asdh](/assets/images/php1/Screenshot 2026-03-16 143940.png)

yang ini dari node worker

![asidh](/assets/images/php1/Screenshot 2026-03-16 144015.png)

dan yang terakhir dari node worker1,nah bisa di perhatikan baik baik untuk enpo03 dan enp0s8 
nah bisa di lihat dari ketiga gambar dari semua enp0s3 itu sama sedangkan enpo0s8 itu unik berbeda-beda sama ya begitu lah cara default dari flannel
mencari dari enp0s3 yang dimana enp0s3 itu sama dan bukannya enp0s8 yang merupakan ip unik masing masing dari node,jadinya mysql tidak bisa terhubung dengan php,

### Solve
maka dari itu ubah dari default pencarian dari flannel dari enpo0s3 ke interface yang merupakan ip dari node misal enp0s8,
cek dengan menggunakan 

```
ip addr
```

coba cek bagian dimana ip unik di mana kah interface yang memiliki ip unik dan bukannya NAT,
